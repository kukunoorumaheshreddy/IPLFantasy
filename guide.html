Match Day Workflow

Before Before the Match Starts

Do this 5–10 minutes before the first ball is bowled.

Open the fantasy website

Go to fantasy.iplt20.com and make sure you're logged in.

Open browser DevTools

Press F12 → click the Console tab.

Start the live polling script

Copy the entire contents of live-poll.js and paste into the console. Press Enter.

You'll see: [LivePoll] Found live match: GD 8 — RCB vs MI

Leave the tab open

The script captures a snapshot every 10 minutes automatically. Don't close the tab or put the laptop to sleep.

If no match is live yet, the script will say "No live match found." Just wait and try again once the match starts.
This step is optional. It only captures intra-match progression. If you skip it, you'll still have the final match data.

During During the Match

Nothing to do! Just keep the browser tab open.

  • The console will log each snapshot: [LivePoll] Snapshot #3 at 2026-04-04T15:30:00Z
  • You'll see the current leader and total points at each capture
  • If the match ends, the script auto-stops and downloads the snapshot file

If your session expires during polling

Don't worry — all snapshots are saved in localStorage automatically. You'll see:

[LivePoll] ⚠️ SESSION EXPIRED! Snapshots are safe in localStorage.
[LivePoll] ➡️ Re-login to fantasy.iplt20.com, then paste this script again.

Just re-login and paste the script again. It will find the saved snapshots and resume from where it left off.

The browser tab title will change to "⚠️ SESSION EXPIRED" so you'll notice even if the console is closed.

Type this in the console:

stopPolling()

It will immediately download all collected snapshots as a JSON file.

After After the Match Completes

Do this once the match result is final (usually 30 min after the match ends to let points settle).

Open fantasy.iplt20.com (if not already open)

Make sure you're logged in. Open F12 → Console.

Run the update script

Copy the entire contents of update-data.js and paste into the console. Press Enter.

Load your existing data (if you have it)

A file picker will appear. Select your existing ipl-fantasy-master.json.

If this is your first time, just wait 30 seconds or press Cancel — it'll fetch everything from scratch.

Wait for it to finish

The script only fetches new matches (skips ones already in your file). It takes ~30 seconds per new match.

You'll see: [Update] ✅ Done! Downloaded: ipl-fantasy-master-gd8.json

View the dashboard

Open dashboard-v2.html (or dashboard.html) in your browser. Drag & drop or click to load your latest ipl-fantasy-master-gdN.json file.

Each run creates a new file like ipl-fantasy-master-gd8.json. Always load the latest one in the dashboard. Old files are your natural backups — no need to manage them.
If the match was still in progress when you extracted, re-run update-data.js after it finishes — it will automatically re-fetch any previously live match with final scores.

🔧 Troubleshooting

"No completed gamedays" or empty data

You're not logged in. Go to fantasy.iplt20.com, log in, then run the script again in the same tab.

File picker didn't appear (update-data.js)

Your browser may have blocked the popup. Just wait 30 seconds — the script will proceed without existing data and fetch everything fresh.

Points don't match the fantasy app exactly

Small differences are expected due to boosters (power-ups some players activate). The script doesn't account for those yet. Differences should be small (under 50 pts).

"stopPolling is not defined"

The page was refreshed or the script wasn't running. But your snapshots are safe in localStorage — just paste the script again and it will pick them up. Then call stopPolling().

Dashboard shows flat lines (same rank every match)

You loaded old data from the v1 script. Re-run update-data.js to get proper per-match data, then reload the dashboard.

🧠 How It Works (Technical)

For the curious — here's what the scripts do under the hood.

update-data.js

  1. Loads your existing ipl-fantasy-master.json (if provided)
  2. Fetches the match schedule from the IPL Fantasy API to find completed matches
  3. Compares with your existing data — only fetches new matches
  4. For each new match: gets every player's fantasy points, then each league member's team composition
  5. Calculates team scores: captain ×2, vice-captain ×1.5, rest ×1
  6. Rebuilds cumulative rankings across all matches
  7. Downloads the merged master file

live-poll.js

  1. Auto-detects the current live match
  2. Every 10 minutes: fetches player points + each member's team → calculates live scores
  3. Stores all snapshots in browser memory (no repeated downloads)
  4. When you call stopPolling() or the match ends, downloads one JSON with all snapshots

Dashboard

Pure HTML + JavaScript (Chart.js). No server needed — just open the HTML file and load your JSON. Works offline.
